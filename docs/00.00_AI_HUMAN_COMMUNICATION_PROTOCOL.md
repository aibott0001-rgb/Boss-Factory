# 🤝 AI-HUMAN COMMUNICATION PROTOCOL
### *Standardized Interaction Rules for Maximum Efficiency & Zero Confusion*

**Version:** 1.0 (Genesis)  
**Status:** 🟢 **ACTIVE MANDATE**  
**Applies To:** All AI Agents (LLMs) and the CEO (Human)

---

## 1. CORE PHILOSOPHY
**"One Action, One Confirmation, Zero Ambiguity."**
To prevent errors, context loss, and wasted time, all interactions must follow a strict request-response-validate loop.

---

## 2. THE GOLDEN LOOP (Mandatory Workflow)

Every task must follow this 4-step cycle:

### Step 1: AI Proposes (Single Action)
- The AI analyzes the goal and proposes **ONE** specific, atomic action.
- **Format:** "To achieve [Goal], please execute this single command: `[Command]`"
- **Constraint:** Never provide multiple complex steps in one message.

### Step 2: Human Executes & Reports
- The Human runs the **exact** command in their terminal.
- The Human copies the **full terminal output** (success or error) and pastes it back.
- **Constraint:** Do not summarize. Paste raw output.

### Step 3: AI Validates
- The AI analyzes the raw output.
- **If Success:** AI confirms ("✅ Step Complete") and proposes the *next* single action.
- **If Error:** AI analyzes the specific error message, checks `docs/17_ERROR_FORENSICS_LOG.md` (if applicable), and proposes a *single* fix command.

### Step 4: Repeat Until Done
- Loop continues until the high-level task is finished.

---

## 3. RULES FOR AI AGENTS

### Rule 1: Stop & Wait
- **NEVER** generate the next step until the Human explicitly confirms the previous one is done.
- **NEVER** assume success without seeing terminal output.

### Rule 2: Context First
- Before starting any new session, the AI **MUST** read:
  1. `docs/11_PROGRESS_STATE_KERNEL.md` (Current Status)
  2. `docs/16_AI_HUMAN_COMMUNICATION_PROTOCOL.md` (This File)
  3. `docs/17_ERROR_FORENSICS_LOG.md` (Known Errors)

### Rule 3: Error Forensics
- If an error occurs, the AI must:
  1. Identify the exact error line.
  2. Check if it exists in `17_ERROR_FORENSICS_LOG.md`.
  3. If known, apply the stored fix.
  4. If new, log it to `17_ERROR_FORENSICS_LOG.md` before proposing a fix.

### Rule 4: No Hallucinations
- Do not invent file paths, function names, or API endpoints. Verify against existing code (`ls`, `grep`) if unsure.

---

## 4. RULES FOR THE CEO (HUMAN)

### Rule 1: Full Output Transparency
- Always paste the **complete** terminal output. Truncated logs hide critical clues.

### Rule 2: One Thing at a Time
- Do not batch multiple requests ("Do X, then Y, then Z"). Let the AI guide you step-by-step.

### Rule 3: Explicit Confirmation
- Use clear keywords: "Done", "Success", "Error: [paste output]", or "Stop".

---
