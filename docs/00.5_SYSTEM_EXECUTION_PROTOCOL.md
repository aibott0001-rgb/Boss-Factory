3. 🛡️ THE GUARDIAN SCRIPT (scripts/guardian.sh)
The Guardian is the automated enforcer of quality. It is not optional.
3.1 Core Functions
Build Test: Runs npm run build. If it fails, nothing is committed.
Error Forensics: Automatically captures error logs and appends them to 17_ERROR_FORENSICS_LOG.md with timestamp and category.
Smart Commit: Generates semantic commit messages based on changed files (e.g., feat(vault): ...).
Progress Sync: Updates the "Last Updated" timestamp in 11_PROGRESS_STATE_KERNEL.md.
Audit Trail: Logs success/failure to 13_EXECUTION_LOG_AUDIT.md.
3.2 Error Handling Logic
When the Guardian detects a failure:
Categorize: Is it Code, Database, Network, or Dependency?
Log: Create a new entry in 17_ERROR_FORENSICS_LOG.md.
Halt: Stop the push process to protect the remote repository.
Alert: Display the error to the CEO/AI for immediate fixing.
